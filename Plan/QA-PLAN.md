# Plan: Resolve and Execute QA-PLAN.md

## Context

`QA-PLAN.md` was just generated by comparing the current `actor-portfolio` Next.js build against the live Squarespace site at trappedactor.com and the existing `trappedactor-PRD.md`. It contains ~70 findings spanning bugs, gaps from the live site, accessibility issues, and "swank" polish opportunities — far more than belongs in a single execution pass.

Before any code is written, the user wants to be grilled on the plan to (a) prune findings that don't apply, (b) resolve the open questions at the bottom of `QA-PLAN.md`, and (c) decide what actually ships in the next slice of work. This plan file will accumulate those decisions as the conversation proceeds.

## Decisions log

- **D1 — Scope model:** Reorganize by area (component/concern), not by severity. Each phase ships all in-scope severities for one area. Tests for an area land in the same phase as the changes.
- **D2 — Hero asset reality check:** `hero.jpg` is 550KB (not multi-MB as QA-PLAN.md implied). P-01 demoted from "biggest LCP lever" to "still worth compressing." Headshots are 10–25KB each — already fine.
- **D3 — Motion always ships with reduced-motion guard:** Any phase introducing animation/transition includes the `prefers-reduced-motion` handling in the same change. G-08 is not a separate ticket; it is a checklist item on every motion-introducing phase.
- **D4 — Modal, carousel, and hero each rewritten once:** When we open `ReelsPreview.tsx`, `Headshots.tsx`, or `Hero.tsx`, all in-scope items for that file land together to avoid double-edits.
- **D5 — Contact form (C-03) → Future.** Mailto only for now. Contact phase becomes a thin polish pass (animated underline, copy-to-clipboard, optional location subtext) on the existing email link. Real form is a separate, user-decided future phase.
- **D6 — Headshot carousel kept (HS-10).** Polish in scope: a11y (keyboard arrows, aria-live, aria-hidden glyph wrapping), simple crossfade, priority tuning. **Out of scope → Future:** "glide-feel" carousel redesign (peek of prev/next, easing curves, swipe-driven momentum).
- **D7 — R-08 Reel dates dropped.** Tiles show title only. R-09 (role/genre tag) remains skipped — no fabricated data.
- **D8 — S-04 Stats kept at PRD 4 fields.** No expansion this round.
- **D9 — C-04 Representation block skipped.** No agent to credit.
- **D10 — N-09 Brand monogram skipped.** Stays as plain "Smaran Harihar" text.
- **D11 — Phase ordering: foundation → visitor eye-path → perf/tests/deploy.** See "Final scope" below.
- **D12 — Animation library: Framer Motion.** Used for entrance animations, headshot crossfade, mobile nav overlay, hamburger morph. All wrapped to honor `prefers-reduced-motion`.
- **D13 — Deploy items in scope.** D-01 / D-02 / D-03 / D-04 land in the final phase of this cycle.

## Verification

Every phase finishes with the same gate before being declared done:

1. `npm run lint && npm run typecheck && npm run test` all pass.
2. `npm run dev` and walk the modified surface manually:
   - Phase 1: skip link reachable on Tab; reduced-motion respected (`System Settings → Accessibility → Display → Reduce motion`); favicon shows in tab; `view-source` shows OG/JSON-LD; anchor clicks land below nav.
   - Phase 2: scroll past 30px → glass nav appears; Tab through links shows focus rings; mobile menu Esc closes + restores focus; active-section underline updates on scroll.
   - Phase 3: hero hits LCP < 2.5s in Lighthouse mobile profile; no layout shift on Safari mobile (test in Responsive Design Mode); Ken-Burns runs unless reduced-motion is on.
   - Phase 4: about image renders crisp; signature italic confirmed in both Chrome and Safari.
   - Phase 5: each video plays; modal × button visible; clicking inside iframe does not close; body doesn't scroll; Esc closes; axe-core clean on the modal.
   - Phase 6: ← / → keys advance carousel; aria-live announcement seen in VoiceOver/NVDA; crossfade smooth; reduced-motion → instant swap.
   - Phase 7: anchor `/#stats` scrolls there.
   - Phase 8: copy-to-clipboard works; mailto opens default mail client.
   - Phase 9: each social link opens correct profile in new tab; back-to-top scrolls smoothly.
   - Phase 10: `npm run build && npx serve out` previews the static export with optimized images; GitHub Actions workflow green on a draft PR; Lighthouse CI ≥ 95 across all four categories.
3. `git diff` reviewed; commit per phase with conventional-commit subject (`feat(nav): glassy scroll state + a11y polish`, etc).
4. Each phase opens its own PR for granular review.
